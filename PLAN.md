# План разработки — модуль «Перевод в PDF»

## Сделано (v0.6.x)

- SMB-шара, дерево файлов, выбор папок и файлов
- Сборка в один PDF с упорядочиванием
- Нумерация страниц (с какой страницы / с какого числа)
- Проверка записи до конвертации (файл занят в другой программе)
- Предупреждение о перезаписи существующих PDF
- Параллельная обработка PDF, стабильный LibreOffice

---

## Этап 2 — качество конвертации (планируется)

### 1. Определение форматов в редактируемых файлах

- Автоопределение реального формата файла (не только по расширению): DOC/DOCX, XLS/XLSX, ODT, RTF, PDF
- Проверка «файл повреждён / не тот формат» до запуска LibreOffice
- Отображение в UI: иконка и подпись формата в списке выбранных файлов
- Лог ошибок с указанием ожидаемого и фактического формата

**Оценка:** ~8 ч

### 2. Определение рамок в модели и перевод в PDF

- Для CAD (DWG/DXF): распознавание рамок листа (форматки) в модели
- Обрезка/масштабирование вывода по рамке при конвертации в PDF
- Сохранение ориентации листа (альбомная / книжная) по рамке
- Для офисных документов: определение полей страницы, совпадение с ГОСТ-рамкой (базово)

**Оценка:** ~24 ч

### 3. Нумерация в PDF слева направо

- Размещение номеров страниц в порядке чтения LTR (слева → направо, сверху)
- Настройка позиции: правый верхний угол / левый нижний / по шаблону
- Учёт альбомной ориентации: номер не «уезжает» за край листа
- Согласование с полями рамки (не перекрывать штамп)

**Оценка:** ~6 ч

### 4. Определение и перенос стиля текста, толщин линий, полилиний

- **Текст:** шрифт, размер, начертание (жирный/курсив), цвет — при конвертации Office → PDF
- **Линии:** толщина (lineweight), тип (сплошная, штриховая), цвет — для DWG/DXF
- **Полилинии:** сохранение геометрии, ширины и слоя при экспорте в PDF
- Сравнение «до/после» в тестовых файлах; отчёт о потерянных стилях

**Оценка:** ~40 ч
