# План разработки — модуль «Перевод в PDF»

> **Область:** всё ниже — только модуль **«Перевод в PDF»** (`convert-to-pdf`).  
> Репозиторий: [Convert-to-PDF](https://github.com/makeden-art/Convert-to-PDF) · сервис `convert-to-pdf` · UI `/convert` · порт `8084`  
> Не относится к нормоконтролю, печати «Маша», сметам и другим модулям платформы.

---

## Сделано (v0.6.x)

- SMB-шара, дерево файлов, выбор папок и файлов
- Сборка в один PDF с упорядочиванием
- Нумерация страниц (с какой страницы / с какого числа)
- Проверка записи до конвертации (файл занят в другой программе)
- Предупреждение о перезаписи существующих PDF
- Параллельная обработка PDF, стабильный LibreOffice

## Этап 2 — качество конвертации в PDF (планируется)

Все пункты реализуются **внутри convert-to-pdf**: `converter.py`, `cad_converter.py`, `convert_page.html`, API `/api/*`.

### 1. Определение форматов в редактируемых файлах

- Автоопределение реального формата (не только по расширению): DOC/DOCX, XLS/XLSX, ODT, RTF, PDF, DWG/DXF
- Проверка «файл повреждён / не тот формат» до запуска LibreOffice / ODA
- Отображение в UI `/convert`: иконка и подпись формата в списке файлов
- Лог ошибок с указанием ожидаемого и фактического формата

**Оценка:** ~8 ч

### 2. Определение рамок в модели и перевод в PDF

- CAD (DWG/DXF): распознавание рамок листа (форматки) в модели
- Обрезка и масштаб вывода по рамке при конвертации в PDF
- Ориентация листа (альбомная / книжная) по рамке
- Офисные документы: поля страницы, базовая проверка ГОСТ-рамки

**Оценка:** ~24 ч

### 3. Нумерация в PDF слева направо

- Номера страниц в порядке чтения LTR (слева → направо)
- Настройка в UI «Перевод в PDF»: позиция (правый верх / левый низ / шаблон)
- Учёт альбомной ориентации и рамки (не перекрывать штамп)

**Оценка:** ~6 ч

### 4. Определение и перенос стиля текста, толщин линий, полилиний

- **Текст (Office → PDF):** шрифт, размер, начертание, цвет
- **Линии (CAD → PDF):** толщина (lineweight), тип штриха, цвет
- **Полилинии:** геометрия, ширина, слой при экспорте в PDF
- Отчёт о потерянных стилях в логе конвертации

**Оценка:** ~40 ч
